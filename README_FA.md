# راهنمای فارسی SNISPF

### ابزار دور زدن فیلترینگ اینترنت

**SNISPF** یه ابزار سبک و ساده‌ست که کمکت می‌کنه فیلترینگ اینترنت رو دور بزنی. کارش اینه که وقتی یه سایت رو باز می‌کنی، اسم واقعی سایت رو از چشم فیلترینگ قایم می‌کنه. روی **ویندوز، مک و لینوکس** کار می‌کنه و نیازی به نصب درایور خاصی نداره.

**جدید در نسخه ۱.۴.۰:** اسکنر حالا اتصالات مسدود و رهگیری‌شده رو به‌درستی تشخیص میده. مشکل قبلی که اسکنر به اشتباه یه سایت بلاک‌شده رو به عنوان سالم نشون می‌داد، برطرف شده.

## چطوری کار می‌کنه؟

وقتی یه سایت HTTPS رو باز می‌کنی، مرورگرت یه پیام «سلام» (به اسم **TLS ClientHello**) می‌فرسته که توش اسم سایت به صورت متن ساده نوشته شده. به این اسم میگن **SNI** (نشانگر نام سرور). سیستم فیلترینگ (یعنی **DPI** یا بازرسی عمیق بسته‌ها) این اسم رو می‌خونه و اگه فیلتر باشه، اتصال رو قطع می‌کنه.

**SNISPF** بین اپ تو و اینترنت می‌شینه. اون پیام «سلام» رو می‌گیره و یا **تیکه تیکه‌ش می‌کنه** یا **یه پیام جعلی می‌فرسته** تا فیلترینگ نتونه اسم واقعی سایت رو بخونه. سرور مقصد ولی پیام کامل و درست رو می‌گیره و همه چیز عادی کار می‌کنه.

```
┌──────────┐     ┌─────────┐     ┌─────────┐     ┌──────────────┐
│ اپ تو    ├────>│ SNISPF  ├────>│فیلترینگ │────>│ سرور واقعی  │
│ (مرورگر، │     │ (پروکسی │     │  DPI    │     │ (مثلاً       │
│  v2ray،  │     │  لوکال) │     │         │     │  Cloudflare) │
│  و غیره) │     │         │     │         │     │              │
└──────────┘     └─────────┘     └─────────┘     └──────────────┘
```

### ویژگی مخصوص ایران

توی ایران، DNS مسموم شده (DNS Poisoning)، یعنی وقتی آدرس یه سایت رو وارد می‌کنی، ممکنه به جای آی‌پی واقعی، یه آی‌پی غلط بهت برگرده. به همین خاطر، SNISPF از نسخه **1.3.0** به بعد، **آی‌پی‌های آماده** (seed IPs) داره که از قبل شناخته شده‌ان و نیازی به DNS ندارن. یعنی حتی اگه DNS تو خراب باشه یا مسموم شده باشه، اسکنر می‌تونه مستقیم آی‌پی‌های کلادفلر رو تست کنه.

## شروع سریع

### مرحله ۱: اجرای ابزار

ساده‌ترین راه برای شروع:

```bash
snispf -l 0.0.0.0:40443 -c 104.18.38.202:443 -s cdnjs.cloudflare.com
```

معنی هر قسمت:

| فلگ | کارش چیه | مثال |
|---|---|---|
| `-l` | آدرس و پورتی که SNISPF روش گوش میده | `0.0.0.0:40443` |
| `-c` | آی‌پی و پورت سرور مقصد (یه آی‌پی کلادفلر) | `104.18.38.202:443` |
| `-s` | اسم جعلی سایت (SNI) که به فیلترینگ نشون میده | `cdnjs.cloudflare.com` |

> **نکته مهم:** اگه نمی‌دونی از چه آی‌پی یا SNI استفاده کنی، از **حالت خودکار** (auto) استفاده کن. خودش همه چیز رو پیدا می‌کنه.

### مرحله ۲: اپت رو بهش وصل کن

وقتی SNISPF اجرا شد، مرورگر، V2Ray، Xray یا هر اپ دیگه‌ای رو تنظیم کن که از این آدرس استفاده کنه:

```
Address: 127.0.0.1
Port:    40443
```

تمام! ترافیکت حالا از SNISPF رد میشه و دور زدن فیلتر خودکار انجام میشه.

## حالت خودکار (پیشنهادی)

حالت خودکار ساده‌ترین و بهترین راه استفاده از SNISPF هست. خودش آی‌پی‌های کلادفلر رو اسکن می‌کنه، سریع‌ترین رو انتخاب می‌کنه، پروکسی رو راه میندازه و اگه آی‌پی فعلی بلاک بشه، خودکار یکی دیگه رو جایگزین می‌کنه.

```bash
# اسکن + اجرای پروکسی با جابجایی خودکار
snispf --auto

# اسکن + پروکسی + اسکن مجدد هر ۵ دقیقه
snispf --auto --rescan 300

# حالت خودکار با قوی‌ترین روش دور زدن
snispf --auto -m combined

# حالت خودکار با SNI‌های دلخواه
snispf --auto --sni-list "cdnjs.cloudflare.com,www.speedtest.net,ajax.cloudflare.com"

# حالت خودکار با لاگ‌های کامل (برای دیباگ)
snispf --auto -v
```

### حالت خودکار چیکار می‌کنه:

<div dir="rtl" align="right">

1. **اسکن:** ۱۰۰ آی‌پی کلادفلر رو تست می‌کنه (اول آی‌پی‌های از پیش آماده، بعد تصادفی)
2. **تست:** هر کدوم رو از نظر سرعت TCP، دست‌دهی TLS و پاسخ HTTP بررسی می‌کنه
3. **تأیید:** بررسی می‌کنه که پاسخ واقعاً از کلادفلر باشه (نه صفحه بلاک یا پروکسی مخفی)
4. **انتخاب:** سریع‌ترین آی‌پی تأیید‌شده رو انتخاب می‌کنه
5. **اجرا:** پروکسی رو راه میندازه و ترافیکت رو از اون آی‌پی رد می‌کنه
6. **نظارت:** اگه آی‌پی بلاک بشه، خودکار به بعدی سوییچ می‌کنه
7. **اسکن مجدد:** اگه `--rescan` تنظیم بشه، مرتب لیست آی‌پی‌ها رو آپدیت می‌کنه

</div>

## اسکنر آی‌پی

اسکنر داخلی، آی‌پی‌های کلادفلر رو تست می‌کنه و بر اساس سرعت و در دسترس بودن رتبه‌بندی می‌کنه. برای پیدا کردن آی‌پی‌های «تمیز» که روی شبکه‌ات فیلتر نشدن خیلی مفیده.

### اسکن یک‌باره

فقط اسکن کن و نتیجه رو ببین (بدون اجرای پروکسی):

```bash
snispf --scan
```

### اسکن با تست سرعت دانلود

```bash
snispf --scan --download
```

### اسکن آی‌پی‌های بیشتر

پیش‌فرض ۱۰۰ آی‌پی تست میشه. اگه نتیجه خوبی نگرفتی، بیشتر تست کن:

```bash
snispf --scan --scan-count 500
```

### ذخیره نتایج (کش)

نتایج اسکن رو روی دیسک ذخیره کن تا دفعه بعد سریع‌تر راه بیفته:

```bash
snispf --auto --scan-cache results.json
```

## تنظیمات

### استفاده از فایل کانفیگ

یه فایل کانفیگ پیش‌فرض بساز:

```bash
snispf --generate-config config.json
```

بعد ویرایشش کن و اجرا کن:

```bash
snispf --config config.json
```

### جدول تنظیمات

| تنظیم | توضیح | مقدار پیش‌فرض |
|---|---|---|
| `LISTEN_HOST` | آدرس IP که روش گوش میده | `0.0.0.0` |
| `LISTEN_PORT` | شماره پورت لوکال | `40443` |
| `CONNECT_IP` | آی‌پی سرور مقصد (کلادفلر) | `104.18.38.202` |
| `CONNECT_PORT` | پورت سرور مقصد | `443` |
| `FAKE_SNI` | اسم سایت جعلی (باید پشت کلادفلر باشه!) | `cdnjs.cloudflare.com` |
| `BYPASS_METHOD` | روش دور زدن: `fragment`، `fake_sni` یا `combined` | `fragment` |
| `FRAGMENT_STRATEGY` | نحوه تکه‌تکه کردن: `sni_split`، `half`، `multi`، `tls_record_frag` | `sni_split` |
| `FRAGMENT_DELAY` | تأخیر بین هر تکه (ثانیه) | `0.1` |
| `SCANNER_ENABLED` | فعال‌سازی اسکنر (مثل `--auto`) | `false` |
| `SCANNER_COUNT` | تعداد آی‌پی‌ها برای تست در هر اسکن | `100` |
| `SNI_DOMAINS` | لیست دامنه‌های SNI سفارشی | `[]` (از لیست داخلی استفاده میشه) |

## روش‌های دور زدن فیلترینگ

### `fragment` (پیش‌فرض)

پیام «سلام» TLS رو به چند تکه تقسیم می‌کنه تا فیلترینگ نتونه اسم سایت رو از یه تکه بخونه.

**بهترین برای:** اکثر مواقع. همه‌جا کار می‌کنه، نیاز به دسترسی خاصی نداره.

```
عادی:   [پیام کامل: ...SNI=سایت-فیلتر-شده...]  --> فیلترینگ بلاکش می‌کنه

SNISPF:  [تکه ۱: ...SN]                        --> فیلترینگ اسم ناقص می‌بینه
         [تکه ۲: I=سایت-فیلتر-شده...]           --> دیگه دیر شده، رد شده
```

### `fake_sni`

یه پیام «سلام» جعلی با اسم یه سایت مجاز می‌فرسته که فیلترینگ اون رو می‌خونه ولی سرور نادیده‌ش می‌گیره.

**بهترین برای:** وقتی fragment تنها کار نمی‌کنه. با root خیلی مؤثرتره.

**با root (لینوکس):** از raw socket استفاده می‌کنه و پیام جعلی رو با شماره ترتیب TCP خارج از پنجره دریافت سرور می‌فرسته. فیلترینگ می‌بینتش و اجازه میده. سرور ولش می‌کنه چون شماره ترتیبش خارج از محدوده‌ست.

**بدون root:** برمی‌گرده به روش fragment.

### `combined` (قوی‌ترین)

هر دو روش رو همزمان استفاده می‌کنه: پیام جعلی (اگه root باشه) + تکه‌تکه کردن پیام واقعی.

**بهترین برای:** فیلترینگ‌های سخت‌گیرانه. مؤثرترین گزینه.

```bash
# روی لینوکس با sudo برای تأثیر کامل:
sudo snispf --auto -m combined --rescan 300
```

## استراتژی‌های تکه‌تکه کردن

| استراتژی | چیکار می‌کنه | کِی ازش استفاده کن |
|---|---|---|
| `sni_split` | دقیقاً وسط اسم سایت رو قطع می‌کنه | پیش‌فرض و مؤثرترین |
| `half` | پیام رو نصف می‌کنه | جایگزین ساده اگه sni_split کار نکرد |
| `multi` | به تکه‌های ۲۴ بایتی تقسیم می‌کنه | برای فیلترینگ‌هایی که دو تکه رو جمع می‌کنن |
| `tls_record_frag` | چند رکورد TLS معتبر از یه پیام می‌سازه | برای فیلترینگ‌هایی که TLS می‌فهمن ولی چند رکورد رو هندل نمی‌کنن |

## عیب‌یابی

### "Permission denied" موقع اجرا

پورت‌های زیر ۱۰۲۴ نیاز به root دارن. از پورت بالاتر استفاده کن:

```bash
snispf -l :40443 ...
```

### "Address already in use"

یه چیز دیگه از اون پورت استفاده می‌کنه. یه پورت دیگه امتحان کن:

```bash
snispf -l :50443 ...
```

### وصل میشه ولی کار نمی‌کنه

به ترتیب این‌ها رو امتحان کن:

1. **از حالت خودکار استفاده کن:**
   ```bash
   snispf --auto -m combined
   ```

2. **روش دور زدن رو عوض کن:** `fragment` -> `combined`
   ```bash
   snispf --auto -m combined
   ```

3. **استراتژی تکه‌تکه کردن رو عوض کن:**
   ```bash
   snispf --auto --fragment-strategy multi
   ```

4. **تأخیر بین تکه‌ها رو زیاد کن:**
   ```bash
   snispf --auto --fragment-delay 0.2
   ```

5. **SNI دیگه‌ای امتحان کن (فقط سایت‌هایی که پشت کلادفلر هستن):**
   ```bash
   snispf --auto --sni-list "cdnjs.cloudflare.com,www.speedtest.net,ajax.cloudflare.com"
   ```

6. **آی‌پی بیشتری اسکن کن:**
   ```bash
   snispf --auto --scan-count 500
   ```

### اسکنر هیچ آی‌پی کار‌آمدی پیدا نمی‌کنه

معمولاً یعنی شبکه‌ات ترافیک کلادفلر رو خیلی سنگین فیلتر می‌کنه:

1. SNI‌های مختلف امتحان کن:
   ```bash
   snispf --scan --sni-list "cdnjs.cloudflare.com,www.speedtest.net"
   ```

2. تایم‌اوت رو زیاد کن:
   ```bash
   snispf --scan --scan-timeout 8
   ```

3. تعداد بیشتری اسکن کن:
   ```bash
   snispf --scan --scan-count 500
   ```

## نکات مهم برای ایران

### مشکل DNS Poisoning

<div dir="rtl" align="right">

توی ایران، DNS مسموم شده. یعنی وقتی مثلاً آدرس `cdnjs.cloudflare.com` رو DNS حل می‌کنه، ممکنه آی‌پی غلط برگردونه. به همین خاطر:

</div>

<div dir="rtl" align="right">

- **از حالت خودکار استفاده کن** (`--auto`). اسکنر SNISPF نیازی به DNS نداره و آی‌پی‌های از پیش آماده رو مستقیم تست می‌کنه.
- **از SNI‌های کلادفلر استفاده کن.** دامنه‌هایی مثل `cdnjs.cloudflare.com` و `ajax.cloudflare.com` چون زیرساخت خود کلادفلر هستن، خیلی کم بلاک میشن.
- **فقط سایت‌هایی که پشت کلادفلر هستن رو به عنوان SNI استفاده کن!** سایت‌هایی مثل `dl.google.com` یا `fonts.googleapis.com` پشت کلادفلر نیستن و کار نمی‌کنن.

</div>

### تشخیص صفحات بلاک و پروکسی‌های مخفی

<div dir="rtl" align="right">

در نسخه ۱.۴.۰ یه مشکل مهم برطرف شده: قبلاً اسکنر فقط بررسی می‌کرد که آیا اتصال TCP و دست‌دهی TLS موفق شده یا نه. این کافی نبود چون:

</div>

<div dir="rtl" align="right">

- فیلترینگ ممکنه دست‌دهی TLS رو اجازه بده ولی بعدش صفحه بلاک نشون بده
- یه پروکسی مخفی (MITM) ممکنه TLS رو قطع کنه و گواهی جعلی بده
- اتصال ممکنه بعد از ارسال داده‌ها ریست بشه

</div>

<div dir="rtl" align="right">

حالا اسکنر بعد از TLS، یه درخواست HTTP می‌فرسته و بررسی می‌کنه که جواب واقعاً از کلادفلر باشه. همچنین گواهی TLS رو چک می‌کنه تا مطمئن بشه یه پروکسی مخفی وسط راه نیست.

</div>

### بهترین تنظیمات برای ایران

```bash
# ساده‌ترین - خودش همه کار رو انجام میده
snispf --auto

# قوی‌تر - combined + اسکن مرتب
snispf --auto -m combined --rescan 300

# با کش (دفعه بعد سریع‌تر شروع میشه)
snispf --auto -m combined --rescan 300 --scan-cache results.json

# اگه لینوکس داری و root هستی (بهترین حالت)
sudo snispf --auto -m combined --rescan 300
```

### نحوه استفاده با V2Ray / Xray

1. اول SNISPF رو اجرا کن:
   ```bash
   snispf --auto -m combined
   ```

2. بعد توی تنظیمات V2Ray/Xray، آدرس سرورت رو به این تغییر بده:
   ```
   Address: 127.0.0.1
   Port:    40443
   ```

3. SNISPF خودش بقیه کارها رو انجام میده - آی‌پی بهینه رو پیدا می‌کنه، فیلترینگ رو دور می‌زنه و اگه آی‌پی بلاک بشه خودکار عوض می‌کنه.

### نحوه استفاده با مرورگر

اگه می‌خوای مستقیم از مرورگر استفاده کنی، نیاز به یه proxy extension داری:

1. **SNISPF رو اجرا کن:**
   ```bash
   snispf --auto
   ```

2. **یه اکستنشن پروکسی نصب کن** (مثل SwitchyOmega یا FoxyProxy) و تنظیمش کن روی:
   ```
   SOCKS5 or HTTP Proxy
   Address: 127.0.0.1
   Port:    40443
   ```
